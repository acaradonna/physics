Wiki

Determinism

APE uses fixed‑step integration and stable ordering to produce deterministic results across runs on the same platform. This is essential for replay, rollback, and robust testing.

Data‑Oriented Design

APE stores bodies in a structure‑of‑arrays layout for cache locality and SIMD‑friendly processing. Handles, not pointers, reference entities for stability.

Broadphase

APE starts with a naive broadphase for correctness, moving towards SAP/BVH. Broadphase identifies potentially colliding pairs efficiently.

Bindings

A stable C ABI enables language bindings and WebAssembly builds for the browser. These demos auto‑use WASM if available; otherwise a JS fallback emulates the API.
